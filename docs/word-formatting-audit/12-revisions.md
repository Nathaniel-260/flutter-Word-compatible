## חלק א' — הייחוס (העתקה מדויקת מהמסמך הראשי)

כשמעקב שינויים פעיל, Word עוטף תוכן/מאפיינים באלמנטי revision. כולם נושאים `@w:author`,`@w:date`,`@w:id`.

| אלמנט | מה עושה | רינדור |
|---|---|---|
| `w:ins` | טקסט/ריצות **שנוספו** | להציג (אולי בקו‑תחתון/צבע אם "show markup") |
| `w:del` | טקסט **שנמחק** (התוכן ב‑`w:delText` במקום `w:t`) | להסתיר (final) או קו‑חוצה (markup) |
| `w:moveFrom`/`w:moveTo` | טקסט שהוזז | |
| `w:rPrChange` | מאפייני ריצה הקודמים (לפני שינוי) | |
| `w:pPrChange` | מאפייני פסקה קודמים | |
| `w:tblPrChange`/`w:trPrChange`/`w:tcPrChange`/`w:sectPrChange` | מאפיינים קודמים של טבלה/שורה/תא/מקטע | |
| `w:numberingChange` | שינוי מספור (legacy) | |
| `w:cellIns`/`w:cellDel`/`w:cellMerge` | שינויי תא | |

> **קריטי:** במצב "final" (תצוגת קריאה רגילה) — מרנדרים `w:ins` רגיל ו**מתעלמים מתוכן `w:del`**. במצב "show markup" מציגים את שניהם בעיצוב מעקב. `w:delText` הוא **טקסט אמיתי** שאסור לרנדר ב‑final.

### ב.2 — פערים והוראות ל‑AI הבא

**החלטת התכן (לתעד כסטייה מודעת — מקובלת לתצוגת קריאה):**
- **המנוע מממש אך ורק את מצב "Final".** ins/moveTo מוצגים, del/moveFrom/delText מוסתרים — נאמן 1:1 לתצוגת ברירת‑המחדל של Word ("Final"/"No Markup"). אין **show markup** (פריט 11): שום markup ויזואלי של שינויים אינו מרונדר, ולכן `@w:author`/`@w:date`/`@w:id` (פריט 1) ו‑`rPrChange`/`pPrChange`/`*PrChange` (פריטים 5‑7) — שנחוצים רק ל‑show‑markup — לא נקראים. **אין צורך בפעולה** אלא אם יתווסף מצב מעקב‑שינויים מלא; אז יש לבנות שכבת רינדור (קו‑חוצה/קו‑תחתון/צבע‑מחבר/בלונים) ולחווט את קריאת המטא‑דאטה והמאפיינים הקודמים.

**פערים אמיתיים בתצוגת final (לשקול תיקון):**
- **שורת טבלה מחוקה עדיין מוצגת (פריט 9).** מחיקת שורה שלמה מסומנת ב‑`w:trPr/w:del`, אך פַּרסר ה‑`trPr` לא קורא אותו → השורה מרונדרת ב‑final למרות שב‑Word היא נעלמת. לתקן: אם `trPr.getElement('w:del') != null` — להשמיט את השורה (וב‑`w:trPr/w:ins` להבטיח הצגה). באותו אופן `cellDel`/`cellMerge` ברמת `tcPr` אינם מיושמים (השפעה מינורית יותר).
- **`moveTo` ברמת בלוק בתוך תא אובד (פריט 4).** `_parseCell` מכיר רק `['ins','del','smartTag','sdt']` → תוכן שהוזז (block‑level `w:moveTo`) בתוך תא נופל בשתיקה. לתקן: להוסיף `moveTo` (פתיחה) ו‑`moveFrom` (השמטה) לרשימה ב‑`table_parser.dart:472`, בהתאמה לטיפול ב‑`block_parser`.

**אי‑עקביות זניחה (תיעוד בלבד):**
- **`del` בתוך תא נפתח במקום מושמט (פריט 3).** `table_parser.dart:472` מאגד `del` עם `ins`. הטקסט עדיין מוסתר (כי `delText` לא נקרא), אך עלולה להיוותר **פסקה ריקה** במקום המחיקה — בעוד שבגוף המסמך המחיקה מושמטת לחלוטין. לעקביות: להוציא את `del` מהרשימה בתא (ולהוסיף טיפול נפרד כמו ב‑`block_parser`).
